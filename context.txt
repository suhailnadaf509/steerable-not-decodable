  Research Proposal: Stage 7 Redesign

  1. The Sharpest Question

  The current framing — "readability-steerability dissociation" — is correct in spirit but needs a different measurement
  strategy to be credible. Here's the sharper version:

  Does the model's own unembedding matrix reveal task-relevant information at intermediate layers for tasks where additive
  FV steering fails? And what, exactly, do the FVs for failing tasks encode?

  This is sharper than "does a learned probe decode the information?" for three reasons:

  First, the logit lens uses the model's own decoding pipeline. It answers "what would the model produce if it decoded at
  this layer?" — which is a stronger claim than "can an external classifier extract this." The Hewitt & Manning (2019)
  critique of probe complexity doesn't apply because there are no learned parameters.

  Second, projecting the FVs themselves through W_U tells us what each FV means in vocabulary space. For successful FVs, we
   expect to see correct output tokens dominating. For failing FVs, we expect to see task-relevant but functionally
  insufficient tokens — sentiment words for sentiment_flip, letter-like tokens for first_letter. This moves beyond
  documenting that something fails to explaining how it fails.

  Third, this approach works for all 12 tasks with zero training. The class-count catastrophe (90 classes, 66 samples)
  disappears entirely.

  2. Why This Matters Right Now

  The activation steering paradigm has become the default tool for behavioral control in the safety community. Zou et al.'s
   representation engineering, Arditi et al.'s refusal direction, Turner et al.'s activation addition — all rest on an
  assumption the field treats as near-axiomatic: if a feature is linearly present in the residual stream, it can be
  linearly steered. Park et al. (2023) articulated the linear representation hypothesis, and the field has largely elided
  the distinction between "representations are linear" and "interventions on representations are linear."

  This experiment is uniquely positioned to break that elision. Stages 1-5 already show that cosine alignment (the standard
   geometric quality metric for steering vectors) has zero predictive power for behavioral outcomes (r=0.013,
  non-significant). But that finding challenges the evaluation method, not the intervention method. The readability
  analysis challenges the intervention method itself: even when the information is there, additive steering may not be the
  right tool.

  The specific prediction this tests: steering failure is not about information absence but about computational
  incompatibility. Retrieval tasks (antonym, country_capital) succeed because "retrieve the associated token" IS a linear
  operation on MLP-stored associations (Geva et al., 2023). Character tasks (first_letter, reverse_word) fail because
  "extract the nth character" requires nonlinear token-to-character routing that no single additive direction can
  implement. Compositional tasks (sentiment_flip) fail because "rewrite with opposite valence" requires multi-token
  generation that transcends residual stream arithmetic.

  The safety implication is concrete: steering-based safety interventions (e.g., "add the refusal direction") are reliable
  for retrieval-like behaviors but potentially unreliable for behaviors requiring compositional reasoning — exactly the
  behaviors safety researchers most want to control.

  3. The Probing Design

  I propose replacing the broken Stage 7 with three analyses:

  Stage 7a: Logit Lens Readability

  For each (task, template, layer, example):
  1. Take the cached zero-shot residual stream activation h at the last token position (already available from Stage 3
  cache)
  2. Apply the model's final layer norm: h_norm = model.ln_final(h)
  3. Compute logits: logits = h_norm @ model.W_U (TransformerLens exposes both)
  4. Tokenize the first token of the expected output
  5. Record: rank of correct token, log-probability, binary top-k membership (k=1, 5, 10)

  Aggregate to get per-(task, template, layer) readability curves. This reuses the existing Stage 3 cached activations — no
   new forward passes needed.

  For sentiment_flip specifically: since the first output token is typically "I" (trivially predictable), we additionally
  compute a sentiment direction score — project the zero-shot activation onto the difference between mean positive and mean
   negative sentiment embeddings (using the task's own input/output pairs as the sentiment corpus). This measures whether
  sentiment polarity is linearly present, which Tigges et al. (2023) already established but we verify on our specific
  model and templates.

  Stage 7b: FV Vocabulary Projection

  For each (task, template, layer):
  1. Take the extracted FV (already computed in Stage 2)
  2. Project through the unembedding: fv_logits = model.ln_final(FV) @ model.W_U
  3. Record the top-50 tokens by logit magnitude (both positive and negative ends)
  4. Compute: what fraction of the top-50 tokens are (a) correct output tokens for this task, (b) plausible but incorrect
  outputs, (c) task-relevant vocabulary, (d) unrelated

  This is the interpretive analysis. For antonym at layer 16, we expect the FV's top tokens to include actual antonyms from
   the training set. For sentiment_flip at any layer, we expect sentiment-valenced words (happy, sad, good, bad) but NOT
  the actual transformed sentences. The gap between "captures the concept" and "implements the function" IS the mechanistic
   explanation for steering failure.

  This analysis follows the tradition of Dar et al. (2022, "Analyzing Transformers in Embedding Space") and Geva et al.
  (2022-2023), but applied systematically to function vectors across a task taxonomy — which hasn't been done.

  Stage 7c: Readability-Steerability Comparison

  The deliverable:
  1. Layer-wise dual curves: logit lens top-5 accuracy vs. FV steering accuracy at each extraction layer, per task. These
  replace Figure 7 in the paper.
  2. Cross-template logit lens consistency: standard deviation of readability across 8 templates at each layer. Compare to
  FV transfer variance.
  3. The populated 2×2 matrix using logit lens readability instead of the broken multiclass probe.
  4. Paired statistical test (Wilcoxon across tasks × templates × layers) with Bonferroni correction.

Okay, here's the honest critique. I'm going to be direct because this paper has real potential but the proposed Stage 7 redesign has some serious blind spots given where the field is in 2026.

**The biggest problem: the logit lens is the wrong tool for your central claim, and the field knows it.**

Sonia Joseph's 2025 post nailed this and it's now widely understood: when the logit lens fails to detect information at a layer, that *doesn't mean the information is absent*. The representations at intermediate layers can be rotated, shifted, or stretched relative to the final unembedding space. You're proposing to build your paper's central figure — the readability-steerability gap — on a tool that systematically underestimates readability at early and middle layers. This means your "readability without steerability" cell could be artificially empty (logit lens says "not readable" when it actually is), and your "both fail" cell could be overpopulated with false negatives.

The proposal acknowledges this indirectly by claiming the logit lens is better than learned probes because "the Hewitt & Manning critique doesn't apply." True, but you've traded one problem for a worse one. Learned probes might overfit and claim information is present when it isn't. The logit lens does the opposite — it *misses* information that IS present because the intermediate representation basis hasn't converged to the output basis yet. For your specific hypothesis (that information IS present but steering fails), you need a readability measure that errs on the side of detection, not the side of conservatism. The logit lens is exactly backwards for your purpose.

**The tuned lens is the obvious fix, but it's not enough either.**

Yes, using Belrose et al.'s tuned lens would correct for the representational drift between layers. It trains a cheap affine probe per layer to minimize KL divergence with the final output distribution. It's more predictive, less biased, and works on models where the logit lens is essentially broken (like BLOOM). The tuned lens library has been updated through late 2025, supports modern architectures, and pre-trained lenses exist for several model families.

But here's the thing — the tuned lens is *still* a linear readability tool. You'd be replacing one linear decoder with a slightly better linear decoder. The December 2025 paper on chaotic dynamics in deep networks showed that steering vectors become unpredictable after O(log(1/ε)) layers due to positive Lyapunov exponents. If that's true, then the readability-steerability gap you're measuring isn't just "information is present but steering can't use it" — it's that the *entire framework of linear intervention is mathematically limited in deep nonlinear systems*. Your paper would be measuring a symptom while missing the disease.

**What you should actually do: three specific changes.**

First, **use both the logit lens AND the tuned lens, and report the gap between them.** The gap itself is informative — it tells you how much representational drift exists at each layer. For tasks where the tuned lens finds information but the logit lens doesn't, you know the information is encoded in a rotated basis. For tasks where neither finds it, you have stronger evidence of genuine absence. Running both is cheap (the tuned lens is just an affine transform per layer), and the comparison makes your readability claims much more robust. You already have the cached activations. Training tuned lens translators on those activations is maybe an hour of GPU time total.

Second, **decompose your FVs through SAEs, not just through the unembedding matrix.** Your Stage 7b (FV Vocabulary Projection) projects FVs through W_U to see what tokens they "point toward." This is fine as far as it goes, but the field has moved past this. The work on SAE decomposition of steering vectors (the LessWrong posts on gradient pursuit decomposition, the EMNLP 2025 paper on SAE feature selection for steering, the SAE-RSV work) shows that you can decompose your FVs into interpretable sparse features and ask: which features are "input features" (encoding what the task IS) versus "output features" (actually steering behavior)? This distinction — which the SAE literature now supports empirically — maps directly onto your readability-steerability distinction but with mechanistic specificity. For your failing tasks like sentiment_flip, you'd expect the FV to contain "input features" (task recognition: "this is about sentiment") but lack the "output features" (the actual computation: "flip the valence and rewrite"). That's a much stronger mechanistic story than "the top-50 tokens are sentiment words but not the actual output sentences."

The practical blocker is that you'd need SAEs trained on your specific models. GemmaScope provides them for Gemma-2-9B. For Llama and Mistral, you'd need to either train your own or use community-trained SAEs (several exist on the Alignment Forum). If that's too expensive, at minimum do the SAE decomposition for one model family as a case study.

Third, and this is the most important one: **connect to attribution graphs / circuit tracing.** Anthropic's 2025 circuit tracing work, now open-sourced and replicated on Gemma-2-2B and Llama-3.2-1B via Neuronpedia, shows that you can actually trace the computational graph from input features through intermediate features to output logits. For your key tasks — especially antonym (where steering works) versus sentiment_flip (where it doesn't) — you could generate attribution graphs and show *why* the computation differs. For antonym, you'd expect a short, direct path from "word identity" features through "associative retrieval" features to the antonym token. For sentiment_flip, you'd expect the path to require multi-step composition that can't be short-circuited by a single additive vector. This would transform your paper from "we measured that X fails" to "we can see inside the model and show WHY X fails."

Now, I realize the Circuit Tracer currently supports smaller models (Gemma-2-2B, Llama-3.2-1B) and you're working with 7-9B models. But even doing this analysis on a smaller model from the same family as a supplementary analysis would be enormously strengthening. It would ground your task taxonomy in actual circuit structure rather than just your ex ante predictions about "computational type."

**On the sentiment_flip specifically:**

Your proposal says you'll compute a "sentiment direction score" by projecting zero-shot activations onto the difference between mean positive and mean negative sentiment embeddings. This is basically re-doing Tigges et al. (2023) on your model. That's fine for verification but it's not novel. What would be novel is showing that the sentiment *feature* exists as an SAE latent (if you have SAEs), that it activates on your sentiment_flip inputs, and that steering on that specific feature alone produces the wrong kind of output (e.g., the model just inserts sentiment words rather than rewriting the sentence). That would be a concrete mechanistic demonstration of "concept ≠ function."

Also, your handling of sentiment_flip's first token ("I") problem is a band-aid. The real issue is that sentiment_flip is a multi-token generation task being evaluated in a framework designed for single-token retrieval tasks. Your whole evaluation protocol (checking if the first output token matches) is fundamentally mismatched to the task. Consider whether you should be using an LLM judge or a sentiment classifier on the full generated output, not just first-token matching. This is a task design problem, not a probing problem.

**On the nonlinear representation hypothesis:**

There's a 2025 OpenReview paper providing theoretical proof that the linear representation hypothesis cannot be universally true, with VC-dimension bounds showing exponential gaps between concept complexity and linear decodability. There's also the FlowSteer work (TMLR, February 2026) showing that nonlinear steering via flow matching achieves 5.4× better distributional alignment than linear steering. The "Sober Look at Steering Vectors" post from the Alignment Forum documented that many behaviors are simply unsteerable via linear methods, and explicitly recommended exploring nonlinear alternatives.

Your paper's framing — "linear readability does not imply linear steerability" — is correct but increasingly obvious. The field is already moving to "linear anything is insufficient for complex behaviors." You need to position against this emerging consensus, not just against the 2023-era assumption that linear readability implies linear steerability. The sharper question is: **for which specific computational types does the linear framework still hold, and where exactly does it break?** That's what your task taxonomy can answer, and that framing is more future-proof.

**On the falsification conditions:**

Your falsification condition #1 ("if logit lens readability tracks FV steering within ±0.05 for ALL 12 tasks, no dissociation exists") is not actually falsifiable with the logit lens because of the conservatism bias. The logit lens will underestimate readability for early/middle layers, creating an artificial gap. You need the tuned lens for this condition to be meaningful.

**Summary of what I'd change:**

Use logit lens AND tuned lens together, report both, use the gap between them as additional signal. Do SAE decomposition of FVs for at least one model family to get mechanistic specificity beyond vocabulary projection. If at all feasible, run attribution graph analysis on a smaller model from one of your families for a couple of key tasks to ground the "computational type" story in actual circuits. Fix the sentiment_flip evaluation to assess full output quality, not just first-token matching. Reframe the paper's central claim to position against the 2025-2026 consensus that linear methods have mathematical limitations in deep networks, rather than against the 2023 assumption that linear readability implies steerability.

The bones of this paper are strong — the task taxonomy, the cross-template design, the Simpson's paradox finding, the 6-model comparison. The Stage 7 redesign just needs to use tools that match the ambition of the questions being asked, and those tools now exist in ways they didn't when the original proposal was drafted.

  information is genuinely absent.

  For sentiment_flip specifically: instead of checking first-token match (which is "I" — trivially predictable), compute a

  model has encoded polarity, even if the first-token metric is meaningless.

  Stage 7b: SAE Decomposition of Function Vectors

  This is the central new analysis. For each (task, template, best_layer):

  1. Load Llama Scope SAEs for Llama-3.1-8B from fnlp/Llama-Scope on HuggingFace. These cover all layers with 32K and 128K

  3. Compute input/output scores for each active feature (following Arad et al., EMNLP 2025):
    - Input score: How much the feature's activation correlates with input properties (does this feature activate when the
  model sees a "task-like" prompt, regardless of output?)
    - Output score: How much ablating this single feature changes the model's output distribution (does removing this
  feature change what the model actually produces?)
    - One forward pass per feature for output scores. With ~10 features per FV × 12 tasks × 8 templates = ~960 forward
  passes. At batch size 128, that's 8 batches — trivial GPU time.
  4. Report per-FV: The fraction of the FV's SAE-decomposition energy in output features vs. input features. Call this the
  output feature ratio (OFR).

  Stage 7c: FV Vocabulary Projection (Retained, Complementary)

  Keep this from the original proposal as a simpler, parameter-free complement. Project FVs through W_U, report top-50
  tokens. This gives immediately interpretable results (readers can see "the antonym FV's top tokens are actual antonyms"
  vs. "the sentiment_flip FV's top tokens are scattered sentiment words"). It costs nothing and aids exposition.

  Stage 7d: Cross-Template SAE Feature Consistency

  For each task, compare the SAE decompositions of FVs from different templates:
  - Do the same features appear? (Jaccard similarity of top-10 feature sets across templates)
  - Are the feature types consistent? (Are they consistently input-type or output-type?)

  This directly explains the cosine alignment paradox: sentiment_flip has cos≈0.991 because the same input features
